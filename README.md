# ⚽ Gyro Soccer Arena

スマホのブラウザで遊べる、**ロケットリーグ風 3D アリーナサッカー**。
ジャイロセンサーで車を操作して、ロケットブースト付きの車でボールを蹴り合うオンラインマルチプレイ対戦ゲームです。

> 兄弟プロジェクト [`racegame`](https://github.com/guttyanneruuuuuu/racegame) のジャイロ入力と車体メッシュをそのまま移植し、サッカー用に物理 / アリーナ / ネットワークを新規実装しています。

---

## 🎮 主な特徴

- **ロケットリーグ風アリーナ**: 床・4側壁・天井に囲まれた長方形のフィールド、両端に縦型ゴール、フィールド全体にブーストパッド配置
- **3D 描画 (Three.js)**: 車・ボール・アリーナをフルでリアルタイム描画
- **ジャイロ操作 (左右)**: 端末を傾けてハンドル操作。`racegame` の自動キャリブレーション・感度調整ロジックをそのまま流用
- **タッチ UI**: ACCEL / BRAKE / JUMP / BOOST の 4 ボタン
- **ロケットブースター**: 燃料消費でターボ加速、地上でも空中でも使える
- **ジャンプ + ダブルジャンプ**: 空中で 2 回目のジャンプでさらに上昇
- **超弾むボール物理**: 車に当たるとめっちゃ飛ぶ ✨、壁・天井・床にも反発
- **オンライン対戦 (PeerJS)**: 1v1 ～ 4v4 まで（最大 8 人 / 部屋コードで合流）
- **ホストオーソリ方式**: 部屋作成者が物理シミュレーションのホストになり、20 Hz で全員に状態を配信
- **ソロ Bot 練習**: 1v1 ～ 4v4 まで Bot 相手に練習可能
- **5 分間 1 試合**: 制限時間内のスコア勝負

## 🕹️ 操作方法

| 操作 | スマホ | PC |
|------|--------|----|
| ステアリング | 端末を左右に傾ける（ジャイロ） | ← / → or A / D |
| アクセル | 画面右下の `ACCEL` | ↑ or W |
| ブレーキ / バック | 画面左下の `BRAKE` | ↓ or S |
| ジャンプ / ダブルジャンプ | 画面右の `JUMP` | Space or J |
| ブースト | 画面右の `BOOST` | Shift |
| ジャイロ感度調整 | 右上の ⚙ アイコン | (PC では不要) |
| ジャイロ基準リセット | 右上の ⟲ アイコン | - |

## 🚀 試合の流れ

1. タイトル画面で **名前 / マシンカラー / チーム** を選択
2. 以下のいずれかを選ぶ:
   - 🏠 **部屋を作る** → 6 文字の部屋コードが発行されるので友達にシェア
   - 🔑 **部屋に入る** → 6 文字の部屋コードを入力
   - 🤖 **Bot と練習** → ソロモードで Bot と対戦
3. ロビーで チーム (🔵 BLUE / 🟠 ORANGE) を選び、ホストが「**⚽ キックオフ！**」
4. 5 分間の試合（制限時間後にスコアの多いチームが勝利）

## 🛠️ 技術スタック

- **Three.js (r128)** — 3D 描画
- **PeerJS (1.x)** — WebRTC ベースの P2P 通信
- **Vanilla JavaScript** — フレームワーク無しの純素な構成

すべてのライブラリは `vendor/` 配下にバンドルされているので、追加のインストール不要。

---

## 🏃 ローカル起動

`python3` などで静的サーバを立てるだけ:

```bash
cd soccer
python3 -m http.server 8000
# http://localhost:8000 にアクセス
```

> iOS でジャイロを使うには `https://` での配信が必要です（GitHub Pages 等で公開してください）。

## 🤖 ホストオーソリ方式について

- **ホスト**（部屋作成者）が、全車の物理シミュレーション・ボール物理・ゴール判定を実行
- 各 **クライアント** は毎フレーム自分の入力 (`steer`, `accel`, `brake`, `boost`, `jump`) をホストへ送信
- ホストは 20 Hz で全車とボールの状態を全員にブロードキャスト
- クライアントは受信状態をローカル予測と Lerp 補間でスムーズに描画

これにより、4v4 = 8 人での同時対戦でも比較的軽量に動作します。

## ⚠️ 既知の制限

- PeerJS の公開クラウドサーバを使用しているため、混雑時に接続が遅延する場合あり
- iOS Safari ではジャイロ利用に明示的な許可が必要（タイトル画面の「許可する」ボタン）
- 端末性能や WebGL ドライバ依存で、ローエンドスマホではフレームレートが低下する場合あり
